根据 P5 Stage-2 的缺口审计结果，在整理最终学习指南时，应采取**保守策略**处理建模部分。指南正文应聚焦于现有的“光栅化/渲染驱动”逻辑，而将缺乏原始素材支撑的几何算法视为“扩展知识”或“自修缺口”。

以下是供撰写 review-iteration 的审计要点及处理建议：

### 1. 最终指南审计要点 (Review-Iteration Points)

*   **职能分离**：确保指南正文仅保留与**曲线光栅化（Rasterization）**和**渲染中的网格细分（Meshing for Rendering）**相关的部分，将 Bezier/B-spline 的数学推导及网格拓扑结构（Half-edge）移至“课程缺口与进阶自学”模块 [1-3]。
*   **概念化处理**：对于 Catmull-Clark 或参数曲面等词汇，仅在“模型表示分类”中作为术语提及，不展开算法细节 [4, 5]。
*   **侧重点转换**：将“建模”学习重点从“如何构造几何体”转向“几何体如何服务于渲染管线”（如坐标变换、可见性处理、纹理参数化） [6-9]。

### 2. 正文应保留的安全范围 (Safe Scope)

*   **参数化曲线基础概念**：保留 $x=f(u), y=g(u)$ 的参数化表示定义，区别于显式函数 $y=f(x)$ [1-3]。
*   **中点细分算法 (Midpoint-test subdivision)**：这是课件明确支撑的曲线绘制（光栅化）启发式方法。通过判断曲线段与弦的距离决定是否继续递归 [2, 3]。
*   **三维模型表示分类**：保留对多边形网格（最通用）、细分曲面（概念）、参数曲面（NURBS，工业标准）、隐式曲面（水滴融合）、点云（SLAM相关）的宏观优劣对比 [5]。
*   **辐射度算法中的自适应网格 (Adaptive Meshing)**：保留为了解决光照阴影边界锯齿而进行的网格细分逻辑，这属于渲染辅助建模 [10, 11]。
*   **网格拓扑常见缺陷**：保留 T-型连接（T-Junction）的概念，作为渲染闪烁瑕疵的原因 [12]。
*   **纹理映射中的参数化 (UV Mapping)**：保留将三维网格“摊平”到二维纹理空间的 UV 参数化概念及其引起的扭曲问题 [9]。

### 3. 正文应避免的内容 (Avoid - 移入缺口说明)

*   **具体样条数学公式**：避免列出 Bezier 曲线的 Bernstein 基函数、B-spline 的递归定义及 de Casteljau 算法的作图过程 [3]。
*   **网格拓扑数据结构**：避免深入 Half-edge（半边）数据结构的指针操作（Next/Prev/Twin/Vertex）及其在几何处理中的优势。
*   **简化与细分算法细节**：避免描述 QEM（二次误差度量）网格简化的折叠逻辑或 Catmull-Clark 细分的具体顶点加权公式 [4, 5]。

### 4. 建议用户补充的 Source 类型

若用户需应对涉及这些缺口的进阶考试或工程需求，建议补充以下资源：
*   **几何处理专项教材**：如《Polygon Mesh Processing》，用于补充 Half-edge 数据结构和网格简化算法（QEM）。
*   **计算机辅助几何设计（CAGB）手册**：用于获取 Bezier 和 NURBS 的完整数学定义。
*   **工业建模软件 API 文档**：如针对参数曲面的 SolidWorks 或 CATIA 相关二次开发资料 [13]。
*   **现代细分曲面论文**：如 Pixar 关于细分曲面在影视渲染中应用的经典文献 [5]。
