在现代光线追踪技术中，为了实现实时渲染，硬件和算法共同构建了一套从宏观到微观的加速体系。以下是对这些核心概念的深入解析：

### 1. AABB Slab Test：区间的交集直觉
**轴对齐包围盒 (AABB, Axis-Aligned Bounding Box)** 是最基础的加速结构，它由三对分别垂直于 $x$、$y$、$z$ 轴的平行平面（称为 **Slab (板片)**）组成 [1]。

**Slab Test (板片测试)** 的直观逻辑是寻找时间的交集：
*   **区间直觉**：当一条光线射出时，我们可以计算它进入和离开每一对平行平面所花费的时间区间 $[t_{min}, t_{max}]$。
*   **交集判定**：光线只有在同时处于三对平面之间时，才算真正进入了包围盒。
    *   我们取这三个时间区间的 **交集**。
    *   判定准则：如果最终进入时间 $t_{enter}$ 小于离开时间 $t_{exit}$，且 $t_{exit} > 0$（即交点在光线前进方向上），则光线与包围盒相交 [1]。
*   **硬件优势**：这种方法只需简单的减法和乘法，非常适合在硬件中固化 [1]。

### 2. BVH 剔除：从 $O(N)$ 到 $O(\log N)$
**层次包围盒 (BVH, Bounding Volume Hierarchy)** 是一种基于物体的树状组织结构 [1, 2]。

*   **暴力遍历的困境 ($O(N)$)**：在没有加速结构的情况下，光线必须与场景中每一个三角形进行求交测试，计算量随物体数量 $N$ 线性增长 [1, 3]。
*   **剔除逻辑 ($O(\log N)$)**：BVH 将场景组织成树状，根节点包围整个场景，叶子节点包含具体的几何体 [1]。
    *   当光线投射时，首先测试它是否与根包围盒相交。
    *   **如果不相交**，则可以瞬间 **剔除 (Cull)** 掉该节点下的数百万个三角形，无需任何后续测试。
    *   **如果相交**，则递归测试其子节点。这种“错过即跳过”的逻辑将搜索复杂度降低到了对数级别 [1]。

### 3. TLAS 与 BLAS：层级化的场景管理
为了平衡静态几何体的高效遍历和动态物体的频繁更新，现代光追 API 将加速结构分为两层 [4]：

*   **BLAS (Bottom-Level Acceleration Structure, 底层加速结构)**：
    *   直接构建在三角形网格之上。它存储物体的具体几何细节（如一个高精度的人物模型）。
*   **TLAS (Top-Level Acceleration Structure, 顶层加速结构)**：
    *   构建在多个物体实例（Instance）的包围盒之上 [4]。
    *   TLAS 存储的是物体的 **变换矩阵**（位置、旋转、缩放）。
*   **直观解释**：如果你在场景中移动一个角色，驱动程序只需要重新构建简单的 TLAS，而不需要重新计算角色复杂的 BLAS [4]。

### 4. 小型场景示例：BVH 如何减少测试
假设场景中有一个 **兔子 (Bunny)** 模型（10万个三角形）和一块 **地板 (Floor)**（2个三角形） [5]。

1.  **不使用 BVH**：每一条射向天空的光线，都要与 100,002 个三角形逐一对比，即使它们都在光线背后。
2.  **使用 BVH**：
    *   场景被分为两个子包围盒：兔子盒和地板盒。
    *   **情况 A**：光线射向天空。它首先测试兔子盒，**不相交**（剔除 10万次测试）；测试地板盒，**不相交**（剔除 2次测试）。总共仅需 **2 次包围盒测试**。
    *   **情况 B**：光线射向兔子。它测试地板盒失败，然后进入兔子盒。此时它只需测试兔子盒内部被细分出的局部节点，而不是全部 10万个三角形 [1]。

### 5. RT Core 与 OptiX：硬件与软件的协作
*   **RT Core (Ray Tracing Core, 光线追踪核心)**：
    *   这是 NVIDIA 显卡中的专用硬件单元。它将原本由软件执行的 **包围盒遍历 (BVH Traversal)** 和 **三角形求交 (Triangle Intersection)** 逻辑固化在电路中 [2, 6]。
    *   它释放了 **CUDA Core (CUDA 核心)**，使其专注于着色逻辑，从而极大地提升了实时渲染帧率 [6]。
*   **OptiX (OptiX Ray Tracing Engine, OptiX 光线追踪引擎)**：
    *   这是一个通用的可编程光线追踪框架 [7, 8]。
    *   **JIT (Just-In-Time, 即时编译)**：它的核心是一个专用编译器，能将用户编写的“光线生成”、“材质着色”等程序片段组合成高性能的 GPU 内核 [7, 9]。
    *   它隐藏了加速结构构建、光线调度等复杂底层细节，让开发者能像使用 OpenGL 编写光栅化管线一样编写光追管线 [10, 11]。
