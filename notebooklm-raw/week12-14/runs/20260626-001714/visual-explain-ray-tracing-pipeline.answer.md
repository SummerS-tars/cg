基于提供的来源，以下是关于 **Whitted 风格光线追踪 (Whitted-style Ray Tracing)** 数据流的可视化解释素材，旨在从原理到实现逻辑展示其递归渲染过程。

### 1. Whitted-style Ray Tracing 数据流逻辑指南

Whitted-style 光线追踪（由 Turner Whitted 在 1980 年提出）是一种逆向模拟光路的方法，其核心在于通过递归生成的“光线树”来计算全局光照信息 [1, 2]。

#### **数据流阶段详解**

1.  **光线生成 (Ray Generation)**
    *   **输入**：相机视点位置、图像分辨率、像素坐标。
    *   **处理**：为每个像素生成一条从视点（眼睛）出发，穿过成像平面像素中心的主光线 (Primary Ray)，也称视线 (Eye Ray) [3, 4]。
    *   **输出**：一条参数化射线 $P(t) = P_0 + tV$ [3]。

2.  **求交测试 (Intersection)**
    *   **输入**：当前光线、场景几何体数据（通常通过 **BVH(Bounding Volume Hierarchy，层次包围盒)** 等加速结构组织）[5, 6]。
    *   **处理**：遍历场景，寻找与光线相交且距离原点最近的物体表面 [3, 7]。
    *   **输出**：若未命中，调用 **未命中程序 (Miss Program)** 返回背景色；若命中，输出 **最近交点 (Closest Hit)** 的属性（如位置、法线 $N$、材质 $k_d, k_s, k_t$）[8-10]。

3.  **着色计算 (Shading) 与 阴影测试 (Shadow Rays)**
    *   **输入**：交点属性、光源位置。
    *   **处理**：
        *   **局部着色 (Local Shading)**：基于 **Blinn-Phong** 模型计算该点受光源直接照射产生的贡献 [1, 11]。
        *   **阴影测试**：从交点向每个光源发射一条 **阴影光线 (Shadow Ray)**。如果阴影光线在到达光源前被其他物体遮挡，则该点处于阴影中，不计入该光源的漫反射和高光贡献 [8, 12]。
    *   **输出**：该像素点的直接光照颜色分量。

4.  **递归循环 (Reflection/Refraction Recursion)**
    *   **输入**：入射方向 $V$、法线 $N$、材质系数。
    *   **处理**：
        *   **镜面反射 (Specular Reflection)**：若材质具有反射性（如镜子），根据反射定律生成次级反射光线 $R$ [12, 13]。
        *   **透明折射 (Refraction/Transmission)**：若材质透明，根据斯内尔定律 (Snell's Law) 生成折射光线 $T$ [12, 13]。
    *   **递归行为**：将生成的新光线重新作为输入，跳转回“步骤 2 (求交测试)”进行递归追踪 [1, 14]。

5.  **递归边界 (Recursion Boundary)**
    *   **停止条件**：
        *   光线未击中任何物体（Miss）[10, 15]。
        *   达到预设的 **最大递归深度 (Max Depth)**，通常为 3 到 5 次，以防止无限循环并平衡计算开销 [15, 16]。
        *   光线贡献权重低于阈值 [17]。

6.  **最终输出 (Output)**
    *   **处理**：将局部光照、递归返回的反射颜色和折射颜色按系数累加：$I = I_{local} + k_r I_{reflection} + k_t I_{refraction}$ [13, 15]。
    *   **输出**：将最终计算的颜色值写入像素的 **帧缓冲 (Frame Buffer)** [18, 19]。

### 2. Mermaid Flowchart 节点与箭头素材

您可以将以下节点逻辑直接改写为 Mermaid 流程图：

*   **[Node: Start]** --> (获取像素坐标)
*   **[Node: RayGen]** (光线生成器: 计算主光线) --> **[Node: Traversal]**
*   **[Node: Traversal]** (场景遍历: BVH 求交)
    *   *If Miss* --> **[Node: Background]** (返回环境背景色)
    *   *If Hit* --> **[Node: Intersection]** (计算交点属性: P, N, UV)
*   **[Node: Intersection]** --> **[Node: LocalShade]** & **[Node: SecondaryRays]**
*   **[Node: LocalShade]** (计算直接光照 + 阴影测试) --> **[Node: Accumulate]**
*   **[Node: SecondaryRays]** (生成反射/折射光线)
    *   *Check Depth* (深度 < Max?)
    *   *If Yes* --> 重新进入 **[Node: Traversal]** (递归开始)
    *   *If No* --> 返回零贡献
*   **[Node: Accumulate]** (颜色加权合并) --> **[Node: WriteBuffer]** (写入输出缓冲)

### 3. 与 P4 光栅化管线 (Rasterization Pipeline) 的主要差别

| 特性 | Whitted 光线追踪 (P6) | 光栅化管线 (P4) |
| :--- | :--- | :--- |
| **处理中心** | **像素中心**：针对每个像素遍历场景物体 [4, 20]。 | **图元中心**：针对每个三角形遍历其覆盖的像素 [21, 22]。 |
| **可见性** | 通过光线求交自然解决，属于图像空间方法 [20]。 | 依赖 **Z-Buffer(深度缓冲)** 算法处理遮挡关系 [23, 24]。 |
| **全局光照** | 递归处理镜面反射和折射，效果物理真实 [1, 15]。 | 难以处理间接光照，需借助环境贴图等“欺骗”视觉的技术 [25, 26]。 |
| **阴影** | 通过发射阴影测试光线生成，逻辑统一 [12, 13]。 | 需额外步骤（如 Shadow Mapping），存在走样问题 [24, 27]。 |
| **计算开销** | 极高，复杂度随几何复杂度和光线深度增加 [13, 28]。 | 相对较低，现代 GPU 对其有极强的硬件流水线优化 [29]。 |
