在 3D 渲染管线中，“投影之后、像素之前”的后段流程主要负责将三维空间中已经确定投影关系的几何体，最终转化为显示器上正确的像素颜色。

### 1. 这一段管线解决的图形问题
这一阶段的核心任务是**可见性（Visibility）**与**离散化（Rasterization）** [1, 2]。它需要解决：
*   **空间剔除**：去除那些虽然在三维空间中存在，但在摄像机视窗（Window）之外的部分 [1]。
*   **坐标映射**：将抽象的数学坐标转换为具体的屏幕像素坐标 [3]。
*   **图元绘制**：将连续的几何形状（如三角形）离散化为由像素组成的图像 [4]。
*   **遮挡处理**：在多个物体重叠时，确定哪一个表面在最前面，从而正确处理遮挡关系 [2, 5]。

### 2. 管线各阶段详解

#### **Clipping（裁剪）**
*   **输入**：经过投影变换后的几何图元数据 [6]。
*   **输出**：仅保留在视窗（Viewing Window）内部的几何图元部分 [1]。
*   **状态**：此时图元处于**裁剪空间（Clip Space）**或**规范化设备坐标（NDC）**附近。
*   **核心逻辑**：通过如 Cohen-Sutherland 或 Sutherland-Hodgeman 等算法，对超出视锥体定义的六个平面的部分进行剔除或截断，必要时会生成新的顶点来重构被切割的多边形 [7, 8]。

#### **Viewport Transformation（视口变换）**
*   **输入**：裁剪后位于 **NDC（Normalized Device Coordinates）** 空间中的 2D 几何图元 [3, 9]。
*   **输出**：位于**图像坐标系（Image Coordinate System / Screen Coordinates）**下的像素位置 [3, 10]。
*   **状态**：坐标从规范化的范围（通常是 $[-1, 1]^3$）映射到具体的屏幕分辨率大小（如 $1920 \times 1080$）[3, 9]。
*   **核心逻辑**：这是一个缩放和平移的映射过程，将 NDC 的 $x, y$ 坐标映射到屏幕的像素宽度和高度上 [10]。

#### **Scan Conversion / Rasterization（扫描转换 / 光栅化）**
*   **输入**：处于屏幕像素坐标下的 2D 几何图元 [1]。
*   **输出**：被图元覆盖的**片元（Fragments）**或像素序列 [1, 4, 11]。
*   **状态**：数据从连续的几何表达转变为离散的、与像素一一对应的状态 [11]。
*   **核心逻辑**：确定哪些像素中心落在三角形或线段内部，并对顶点的属性（如颜色、深度、纹理坐标）进行插值，为每一个覆盖到的像素生成片元数据 [4, 5]。

#### **Hidden Surface Removal（HSR，消隐 / 深度测试）**
*   **输入**：光栅化生成的带有深度信息（$Z$ 值）的片元 [5]。
*   **输出**：最终写入**帧缓冲区（Frame Buffer）**的像素颜色 [12]。
*   **数据状态**：最终确定像素的可见性。
*   **核心逻辑**：现代硬件主要使用 **Z-Buffer（深度缓冲）算法**。它为每个像素维护一个深度值，在绘制新片元时，比较其深度与缓冲区中已有的深度，仅保留距离相机更近的颜色数据 [5, 12]。

### 3. 与 Week 3-4 MVP / NDC 的关系
*   **MVP 承接**：Week 3-4 讨论的模型-视图-投影（MVP）矩阵是此阶段的前提。Projection Transformation（投影变换）将顶点带入裁剪空间 [13, 14]。
*   **NDC 的中转**：在投影矩阵作用后、视口变换之前，系统会执行**透视除法（Perspective Division）**，将齐次坐标映射为 **NDC** [9]。后段管线中的 Clipping 通常在 NDC 之前或过程中进行，而 Viewport Transformation 则是将 NDC 转为实际像素的“最后一跳” [3, 9]。
*   **数据流向**：后段管线直接消费 MVP 产生的几何顶点，将其转化为可供显示器显示的像素信息 [9, 11]。
