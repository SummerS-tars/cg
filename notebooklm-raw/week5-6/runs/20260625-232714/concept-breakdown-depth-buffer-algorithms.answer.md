在计算机图形学中，消隐（Hidden Surface Removal, HSR）算法旨在确定场景中哪些表面是可见的，哪些是被遮挡的。基于课件和相关资料，现对 Part 3 中的消隐算法展开讲解：

### 1. Z-buffer（深度缓冲算法）
Z-buffer 是现代 GPU 硬件中处理消隐的核心算法，属于图像空间（Image Space）算法 [1, 2]。

*   **数据结构**：在图像的颜色缓冲区之外，额外增加一个深度通道（Depth Channel），即 **Z-buffer** [3, 4]。它通常为每个像素存储一个 24 位或 32 位的深度值（Z 值），代表物体距离观察者的远近 [2, 4]。
*   **像素深度比较流程**：
    1.  **初始化**：将 Z-buffer 中的每个像素点初始化为最大深度值（代表无穷远 $Z_{FAR}$），并将颜色缓冲区初始化为背景色 [2, 4, 5]。
    2.  **渲染比较**：对于场景中的每个多边形，计算其覆盖的每个像素点 $p_i$ 处的深度值 $z$ [4, 5]。
    3.  **测试与更新**：将当前计算的 $z$ 与 Z-buffer 中该像素已存的深度值 $Z\text{-buffer}[p_i]$ 进行比较 [2, 4]。如果 $z < Z\text{-buffer}[p_i]$（即当前表面更近），则更新 Z-buffer 值为 $z$，并更新颜色缓冲区为该表面的颜色；否则，丢弃该片元 [2, 4, 5]。
*   **与光栅化/片元生成的关系**：Z-buffer 作用于光栅化阶段。片元（Fragment）生成时，其深度值是通过多边形顶点处的深度值进行 **双线性插值（Bilinear Interpolation）** 得到的 [2, 3, 5]。这种处理方式允许硬件以并行方式对多边形进行光栅化，且多边形的绘制顺序可以是随机的 [4]。

### 2. A-buffer（A 缓冲算法）
A-buffer（Accumulation Buffer）是对 Z-buffer 的扩展，旨在解决 Z-buffer 的局限性 [4]。

*   **补全功能**：Z-buffer 容易产生**锯齿（Aliasing）**，且无法很好地处理**半透明（Transparency）**物体，因为 Z-buffer 仅保留每个像素最靠近观察者的单一深度信息 [2, 4, 6]。
*   **基本思想**：A-buffer 在每个像素位置存储一个**已排序的片元列表（Sorted list of fragments）** [6]。它可以累积多个覆盖同一像素的片元信息，从而支持高质量的抗锯齿和透明度混合处理（先绘制不透明物体，再处理透明物体） [6]。

### 3. 其他消隐算法概述

*   **BSP Tree（二叉空间剖分树）**：
    *   **基本思想**：这是一种预处理场景的空间数据结构。它通过递归地选择多边形作为**分割平面（Splitting Plane）**，将场景空间分割为树状结构 [7-9]。
    *   **可见性确定**：渲染时根据摄像机在树中节点的位置，按特定顺序（如中序遍历的变体）遍历子树，从而为静态场景提供完美的从远到近（或从近到远）的绘制顺序，解决了多边形循环遮挡的排序难题 [8-10]。
*   **Scan-line（扫描线算法）**：
    *   **基本思想**：将消隐问题从二维图像空间简化为一维扫描线处理 [6, 9]。
    *   **机制**：对于每一条扫描线，算法维护一个活动边表，构建**跨度（Spans）**，并在扫描线穿过边缘时通过比较深度来确定哪个表面在最上方 [6, 7, 9]。它能充分利用扫描线的相干性（Coherence） [7, 9]。
*   **Area Subdivision（区域子分算法 / Warnock 算法）**：
    *   **基本思想**：利用图像空间的采样特性，采用“分而治之”的策略 [7]。
    *   **机制**：如果一个像素区域足够简单（例如区域内只有一个多边形，或某个多边形完全遮挡了区域内其他物体），则直接对该区域着色 [7, 11]。否则，将区域递归地划分为更小的四个子区域，直到满足简单条件或达到像素分辨率限制 [7, 11]。

### 4. OpenGL/GPU 中 Depth Test 的位置
在 OpenGL 渲染管线中，**Depth Buffering（深度缓冲）** 是与视角平截面裁剪（View Frustum Clipping）和背面剔除（Back-face Culling）并列的消隐模块 [12]。在标准的 3D 渲染流水线示意图中，消隐决策通常发生在 **Scan Conversion（扫描转换/光栅化）** 过程中或其之后 [1, 4]。现代 GPU 可以在光栅化生成片元后、写入帧缓冲区之前执行深度测试（Depth Test） [2, 5]。
